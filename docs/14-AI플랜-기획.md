# 14. AI 플랜 기획 — "물범에게 부탁하기" 🦭✨

작성일: 2026-07-08 (Ethan 아이디어 + 창의성 가미)
상태: 기획 단계 (구현은 소셜 로그인·배포 이후, docs/12 로드맵 참조)

## 컨셉

AI 기능이 아니라 **"물범에게 부탁하기"** — 사용자가 몇 가지를 고르면 물범이 뒤뚱뒤뚱 나갔다가
계획서를 물고 돌아온다. 결과는 **기존 테마 플랜(docs/04, M3)으로 저장**되어 캘린더·알람·체크리스트가 자동으로 붙는다.

### 왜 구현 비용이 낮은가 (핵심 통찰)
M3에서 플랜을 "이름+항목+날짜+알람"의 일반 도메인으로 만들어둠 →
**AI는 PlanRequest JSON을 대신 채워주는 것뿐.** 테마 추가 = 프롬프트 템플릿 추가.

## 테마별 입력 폼 → AI 생성물

| 테마 | 사용자가 고르는 것 | 물범이 만들어 오는 것 |
|---|---|---|
| ✈️ 여행 | 도시, 날짜, 인원, 동행(연인/친구/가족), 예산감 | 일자별 코스 + 준비물 체크리스트 + 세부 일정 알람 |
| 💪 운동 | 하루 가능 시간, 부위, 기구(맨몸/덤벨/헬스장), 휴식 시간 | 요일별 운동 구성 (세트·횟수·휴식) + 알람 |
| 🍚 식단 | 목표(다이어트/린매스업/벌크업), 목표 칼로리, 단백질 g, 못 먹는 것 | 하루 식단 예시 + 장보기 리스트 |
| 📖 독서 | 책 제목, 총 페이지, 하루 독서 시간, 완독 목표일 | 일일 분량 계획 + 독서 시간 알람 + D-day |
| 📚 공부 | 과목/시험, 시험일, 하루 가능 시간 | 일/주/월 계획 트리 + 뽀모도로 알람 |
| 🧴 세안 | 보유 제품·성분(레티놀, 비타민C, 나이아신아마이드…), 피부 고민 | 아침/저녁 순서 + **성분 주기 배치** (레티놀 화·금 등) |

### 창의 디테일
- **성분 궁합 경고**: "레티놀과 비타민C는 같은 날 겹치지 않게 배치했어요" — 세안 테마의 차별화
- **계획 재조정 (킬러 기능)**: 계획이 밀리면 "3일 밀렸네? 물범이 다시 짜올게" — 실패를 리텐션으로 전환
- **연출**: 요청 → 물범이 화면 밖으로 뒤뚱뒤뚱 → 몇 초 후 계획서 물고 복귀 → 도장 쾅으로 확정

## 아키텍처 (구현 시)

```
앱 → 우리 서버 POST /api/ai/plans (테마+선택지)
      → 서버가 LLM API 호출 (structured output으로 PlanRequest JSON 강제)
      → 기존 PlanService.create() 재사용 → 캘린더/알람 자동
```
- **API 키는 반드시 서버에만** (앱에 넣으면 유출) — 서버 프록시 필수
- 모델: 경량 모델로 충분 (구조화된 JSON 생성 작업)
- 식단·운동: **건강 조언 면책 문구 필수** ("의료 전문가 조언이 아닙니다")

## 가격 정책 (Ethan 질문: "회당 500원 너무 싸나?")

- 원가: 회당 LLM 비용 약 **10~30원** (입력 2K+출력 2K 토큰, 경량 모델) → 500원도 마진 충분. 싸지 않음
- **그러나 회당 현금 결제는 마찰이 큼** (매번 결제창 + 스토어 수수료 15~30%)
- **채택 제안**: 조개 경제 통합 (결정로그 #11 원칙과 일관)
  - 첫 1회 무료 (마법 경험) → 이후 **AI 플랜 1회 = 조개 300개**
  - **Sealo Plus 구독에 월 5회 포함** (구독 가치의 앵커) + 초과분은 조개
  - 현금 환산 시 회당 실효 300~500원 수준으로 Ethan 감각과 일치
