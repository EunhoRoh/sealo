## 2. 검증 미완이지만 복수 소스가 일치하는 것 ⚠️

- **Duolingo 매출 구성: 구독 ~73–76%, 광고는 9–15%에 불과** (2023, 복수 소스 일치). 광고는 보조 수익 + 유료 전환 마중물 역할.
- Duolingo 광고는 **레슨 사이 자연스러운 휴지점에만** 배치, 레슨 도중엔 안 끼움. 레슨 후 광고 1개는 리텐션에 무해, 2개부터 이탈 (A/B 테스트 결과라고 알려짐).
- Duolingo 하트/보석: 하트 소진 = 마찰 → 광고 시청 or 보석 or 구독으로 해소. **핵심 콘텐츠(레슨)는 끝까지 무료**, 유료는 편의(무제한 하트, 광고 제거, 스트릭 복구).
- **Finch (물범과 가장 가까운 벤치마크, 펫 키우기 셀프케어 앱)**: 부트스트랩으로 ARR 수천만 달러 규모 (소스별 $12M~$40M+로 추정 편차 큼 ❌ 정확 수치는 불명). 모델: **핵심 기능 전부 무료 + 유료는 꾸미기/커스터마이징 확장** ($9.99/월, $39.99–79.99/년), 소프트 페이월. 관대한 무료 티어가 오히려 전환을 만든다는 철학. 주 사용자층 25–35 여성.
- 한국 시장 (오픈서베이 2025–2026): 유료 구독자 대부분 예산 없이 필요 기반 구독, **20–30대가 가장 다양한 카테고리에 지갑을 엶**. 구독 수용도 자체는 높은 시장.

## 3. 기각된 주장 ❌ (참고용)

- "소프트 페이월이 하드 페이월보다 2–3배 잘 전환" — 근거 부족으로 기각 (오히려 반대 데이터가 검증됨, 단 위 표의 주의사항 참조)
- Finch 월 $900K 매출 — 추정치 편차가 커서 기각. "정확한 수치는 아무도 모름"이 결론
- Duolingo 전환율 3%→8.8% 성장 스토리 — 검증 실패 (블로그발 수치)

## 4. Sealo 수익화 설계 (제안)

### 대원칙 (Finch + Duolingo 공식)
1. **핵심 루프(루틴→알림→도장)는 영원히 무료.** 루틴 개수 제한 같은 코어 제한 금지
2. 과금은 **감정/꾸미기/편의**에만: 물범 꾸미기, 도장 디자인, 테마, 알람 사운드, 도장 복구
3. 광고는 주 수익원이 아니라 **보조 수익 + "광고 없애려면 구독" 마중물**
4. 구독은 **연간 중심 + 긴 무료 체험(2주+)**, 주간 플랜 금지

### 단계별 로드맵
| 단계 | 시점 | 수익화 | 이유 |
|---|---|---|---|
| Phase 0 (MVP) | 출시 | **과금 없음.** 단, 재화(조개?)+상점 경제는 MVP부터 탑재 | Duolingo도 수익화는 나중. 먼저 리텐션 검증. 경제 구조는 나중에 붙이기 어려우니 처음부터 |
| Phase 1 | 리텐션 확인 후 (D30 잔존 등) | **보상형 광고**: 광고 보면 재화 지급. 루틴 완료 후 휴지점에 1개만 | 광고 1개까지는 리텐션 무해(듀오링고). 보상형은 사용자가 자발 선택이라 거부감 최소 |
| Phase 2 | 유저 기반 형성 후 | **Sealo Plus 구독 (연간 중심)**: 광고 제거, 프리미엄 꾸미기/도장/테마, 도장 복구(스트릭 보험), 상세 통계. + 소모품 IAP(도장 복구권) | 매출의 본체는 구독 (듀오링고 73%+). 스트릭 복구는 검증된 최강 결제 동기 |

### v2+ 아이디어: 폰 잠금 해제 과금 (2026-07-07 Ethan 제안, 설계 보류 중)

제안 원안: 잠금 해제 1회는 무료, 다음부터는 1,000원에 15분 해제.

**평가: 검증된 "커밋먼트 디바이스 과금" 패턴.** 선례 — Beeminder(목표 어기면 실제 과금), 듀오링고 스트릭 복구(실패 순간의 결제), Forest(어기면 나무가 죽는 감정 페널티).

**리스크**: 사용자가 가장 약해진 순간에 현금 요구 → "돈 뜯는 앱" 리뷰 폭탄 위험, 귀여운 브랜드와 충돌. iOS Screen Time API 특수 권한 + Android 접근성 정책 → 구현 난이도 높음.

**다듬은 설계안**:
1. 1차 결제 수단은 **재화(조개)**, 현금은 재화 충전 경로로 (매출 동일, 거부감 최소)
2. 프레이밍은 벌금이 아니라 **"물범과의 약속을 깨는 것"** — 해제 시 물범 슬퍼하는 연출 (Forest 공식)
3. 하루 1회 무료 해제 유지 (원안대로) → 무료·유료 그라데이션
4. **긴급 탈출구 필수**: 전화·긴급 앱은 항상 무료 개방 (스토어 정책 + 윤리)
5. **허용 앱 화이트리스트** (2026-07-07 추가): 잠금 중에도 사용할 앱을 사용자가 직접 설정 가능 (예: 지도, 음악, 은행). 기본값은 전화+문자만 허용
6. 도입 시점: v2 이후, 잠금 기능 자체가 안정된 뒤

### 질문에 대한 직접 답변
- **"광고 추가해야겠지?"** → MVP엔 넣지 말 것. 넣게 되면 배너보다 보상형 광고 우선, 휴지점에 1개만
- **"루틴 상세 기능에 과금?"** → 금지. 코어에 과금하면 습관 형성 전에 이탈 (Finch의 반대 교훈)
- **"알람 음악에 과금?"** → 좋은 직감! 알람 사운드팩(물범 목소리 등)은 "편의/감성" 영역이라 과금 적합. Plus 혜택 또는 개별 IAP로
- **"처음부터 요금제 유도 안 하지?"** → 맞음 (Duolingo 검증 ✅). 단, 재화가 줄어드는 구조(보석/하트류)는 나중에 결제 동기가 되니 **경제 설계는 MVP부터**
