# 09. 소셜 로그인 + JWT 준비

작성일: 2026-07-07
상태: **Ethan의 사전 준비 필요** — 아래 키 발급 후 개발 시작

## 설계 (확정 제안)
- 흐름: 앱에서 카카오/구글 SDK 로그인 → 받은 토큰을 `POST /api/auth/login`으로 전송 → 서버가 카카오/구글에 검증 → 회원 생성/조회 → **자체 JWT(액세스+리프레시)** 발급
- 이후 모든 API는 `Authorization: Bearer` 헤더 (현재 `X-Member-Id` 헤더 대체)
- 라이브러리: 프론트 `@react-native-seoul/kakao-login` + `expo-auth-session`(구글), 서버 Spring Security + jjwt

## Ethan이 준비할 것 (각 10~20분)

### 카카오
1. https://developers.kakao.com → 애플리케이션 추가 (이름: Sealo)
2. 플랫폼 등록: Android 패키지명 / iOS 번들 ID (예: `com.eunhoroh.sealo` — 통일 필요)
3. 카카오 로그인 활성화 + 동의 항목(닉네임)
4. 필요한 값: **네이티브 앱 키**, **REST API 키**

### 구글
1. https://console.cloud.google.com → 새 프로젝트 (Sealo)
2. OAuth 동의 화면 구성 (외부, 테스트 사용자에 본인 이메일)
3. 사용자 인증 정보 → OAuth 클라이언트 ID 생성: **Android / iOS / 웹** 3종
4. 필요한 값: 클라이언트 ID 3종

### 공통
- [ ] 앱 식별자 확정: `com.eunhoroh.sealo` (제안) — 한번 정하면 스토어에서 못 바꿈
- [ ] 발급받은 키는 **저장소에 커밋 금지** — `.env` / 환경변수로 관리 (`.gitignore`에 이미 포함)

준비되면 "로그인 키 준비됐어"라고 알려주면 바로 구현 시작.
