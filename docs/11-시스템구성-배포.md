# 11. 시스템 구성 & 배포

작성일: 2026-07-07
상태: 로컬 개발 환경 ✅ / 클라우드 배포는 **계획** (MVP 기능 완성 후)

## 현재 시스템 구성 (로컬 개발)

```
┌─────────────────────────┐         ┌──────────────────────────┐
│  frontend (Expo SDK 57)  │  HTTP   │  backend (Spring Boot 4.1)│
│  iOS / Android / Web     ├────────▶│  :8080                    │
│  TanStack Query + axios  │  REST   │  JPA / Spring Security    │
│  로컬 알림(expo-notif.)   │         │  (임시 인증: X-Member-Id)  │
└─────────────────────────┘         └────────────┬─────────────┘
                                                 │ JDBC
                                    ┌────────────▼─────────────┐
                                    │  PostgreSQL 16 (Docker)   │
                                    │  :5432  (compose.yaml)    │
                                    └──────────────────────────┘
```

- 알림은 서버를 거치지 않는 **기기 로컬 알림** (결정로그 #16) — 서버 다운과 무관하게 동작
- 테스트는 H2(in-memory)로 DB 없이 실행

## 로컬 실행 방법

```bash
# 1. 백엔드 (요구사항: Docker Desktop, 이후는 자동 — JDK 21도 Gradle이 받아옴)
cd backend
docker compose up -d        # PostgreSQL 16 기동
./gradlew bootRun           # http://localhost:8080 (헬스체크: /actuator/health)

# bootRun이 Gradle 데몬 문제로 도중에 죽는 경우(간헐 발생) — jar 직접 실행이 안정적:
./gradlew bootJar
java -jar build/libs/sealo-backend-0.0.1-SNAPSHOT.jar

# 2. 프론트
cd frontend
npm install
npx expo start              # w=웹 브라우저 / 실기기=Expo Go 앱으로 QR 스캔
```

- 실기기에서 로컬 백엔드 접근: `EXPO_PUBLIC_API_URL=http://<PC의 LAN IP>:8080` 환경변수 설정 (Android 에뮬레이터는 `10.0.2.2`)
- 개발용 회원(id=1)과 상점 카탈로그는 서버 첫 기동 시 자동 시드

## 데이터/백엔드 들여다보기 (Django admin 대체)

Spring Boot는 Django와 달리 **admin 페이지가 내장돼 있지 않다.** 대신:

```bash
# 1. 터미널에서 바로 SQL (도커 컨테이너 안의 psql 사용)
docker exec backend-db-1 psql -U sealo -d sealo -c "\dt"                # 테이블 목록
docker exec backend-db-1 psql -U sealo -d sealo -c "select * from member;"
docker exec -it backend-db-1 psql -U sealo -d sealo                    # 대화형 접속 (\q로 종료)
```

- **GUI 툴 (추천)**: [DBeaver](https://dbeaver.io/) (무료) 또는 IntelliJ Ultimate의 Database 탭
  - 접속 정보: `localhost:5432` / DB `sealo` / user `sealo` / pw `sealo-local`
- **서버 상태**: http://localhost:8080/actuator/health (헬스), /actuator/metrics (지표)
- **API 직접 호출**: http://localhost:8080/api/shop/items 처럼 브라우저에서 GET API는 바로 확인 가능
- 운영 단계에서 데이터 관리 화면이 필요해지면 별도 백오피스(어드민 API + 웹)를 만든다 — Django처럼 공짜로 생기지 않는 대신 필요한 만큼만 (v2+)

## 클라우드 배포 계획 (제안 — 배포 시점에 확정)

목표: **대기업 이직 포트폴리오** 가치 → AWS 키워드 우선. 단, 사이드 프로젝트라 비용 최소화.

| 구성요소 | 계획 | 비고 |
|---|---|---|
| 백엔드 | AWS **EC2** (t4g.small, ARM) + Docker | 트래픽 생기면 ECS/ALB로 확장 (포트폴리오 스토리) |
| DB | AWS **RDS PostgreSQL** (프리 티어) | 백업 자동화 |
| 배포 파이프라인 | **GitHub Actions**: push → 테스트 → Docker 이미지 빌드 → ECR → EC2 배포 | CI는 지금도 추가 가능 |
| 도메인/HTTPS | Route53 + ACM (또는 초기엔 Cloudflare) | api.sealo.app (도메인 구매 필요) |
| 모바일 배포 | **EAS Build** (Expo 클라우드 빌드) → Play Store / App Store | 웹은 EAS Hosting 또는 Vercel |
| 시크릿 | GitHub Secrets + EC2 환경변수 | 카카오/구글 키, JWT 시크릿, DB 암호 |
