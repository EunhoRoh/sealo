# 02. 기술 스택 (제안)

작성일: 2026-07-07
목표: **대기업 이직에 유리한 스택** + 모바일 앱 출시 + 웹 지원 + 풀스택

## 백엔드: Spring Boot ✅ (질문에 대한 답: 네, 좋습니다)

한국 대기업(네이버, 카카오, 쿠팡, 배민, 토스, 금융권, SI 전부)의 서버 채용은 **Java/Kotlin + Spring** 이 압도적 주류. 커리어 목적이면 Spring Boot가 정답.

| 항목 | 선택 | 이유 |
|---|---|---|
| 언어 | Java 21 (또는 Kotlin) | 대기업 공고 대부분 Java. Kotlin도 카카오/토스 등에서 수요 증가 |
| 프레임워크 | Spring Boot 3.x | 표준 |
| ORM | Spring Data JPA + QueryDSL | 한국 채용 면접 단골 주제 |
| DB | PostgreSQL (또는 MySQL) | 둘 다 무난. MySQL이 국내 기업 점유율은 더 높음 |
| 인증 | Spring Security + JWT + 소셜 로그인(카카오/구글) | 실무 필수 요소 |
| 인프라 | AWS (EC2 or ECS + RDS) | 대기업 공고 필수 키워드 |
| 푸시 알림 | FCM (Firebase Cloud Messaging) | iOS/Android 통합 푸시 표준 |
| CI/CD | GitHub Actions | 무료 + 보편적 |

## 프론트: React Native + Expo ✅

**모바일(iOS/Android) + 웹을 하나의 코드베이스로** 커버 가능. 질문 주신 포인트에 대한 답:

- **"웹으로도 되면 빠르게 구현 가능하지 않을까?"** → 맞음. Expo는 `npx expo start` 후 **w** 키 하나로 브라우저에서 바로 확인 가능.
- **"모바일은 에뮬레이터를 계속 돌려야 하지 않아?"** → 아님. **Expo Go** 앱을 본인 폰에 깔면 QR 스캔으로 실기기에서 즉시 실행 + 코드 저장하면 실시간 반영(Fast Refresh). 평소엔 웹/실기기로 개발하고, 에뮬레이터는 스토어 제출 전 확인용 정도.
- 커리어 관점: React Native = React이므로 **웹 프론트(React) 역량으로 그대로 인정됨**. 국내 프론트 채용의 주류가 React.

| 항목 | 선택 |
|---|---|
| 프레임워크 | React Native + Expo (TypeScript) |
| 라우팅 | Expo Router |
| 상태관리 | Zustand (가볍고 심플) + TanStack Query (서버 상태) |
| 스타일 | NativeWind (Tailwind 문법) |

### 대안 비교 (참고)
- **Flutter**: 앱 완성도/성능 좋지만 Dart는 국내 대기업 웹 채용과 연결 안 됨 → 탈락
- **네이티브 (Kotlin/Swift)**: 두 벌 개발 + 웹 별도 → 사이드 프로젝트로 과함
- **Next.js 웹 + 별도 앱**: 코드 두 벌 → 과함
